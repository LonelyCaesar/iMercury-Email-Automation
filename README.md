# iMercury－整合人工智慧應用之電子郵件自動管理系統

**iMercury** 是國立臺北商業大學資訊管理系人工智慧科技應用學士後專班的團隊專題研究。

本專題針對企業電子郵件仰賴人工閱讀、分類、轉寄與通知所造成的行政負擔，研究如何整合自然語言處理、生成式 AI、影像理解與流程自動化技術，建立一套電子郵件自動管理流程。

系統利用 **n8n** 串接 Gmail、OpenAI、Google Sheets、Python 與 LINE Messaging API，完成郵件擷取、內容分析、部門分類、收件信箱查詢、自動轉寄及即時通知；研究過程亦包含郵件分類模型與影像描述模型的實驗及評估。

> 本專案定位為「團隊研究型技術實作專題」，兼具研究分析、模型實驗與系統流程整合，並非單一成員獨立完成的商業產品。

## 核心研究問題

傳統企業郵件處理通常需要人工執行下列工作：

1. 閱讀郵件主旨與內文。
2. 判斷郵件所屬部門。
3. 查找正確的部門信箱。
4. 重新整理並轉寄郵件。
5. 通知管理人員或相關承辦人。

本專題研究如何以 AI 與流程自動化技術降低重複性操作，並讓郵件從接收、分析到派送形成較完整的自動化流程。

---

## 系統功能

### 1. 郵件自動擷取

- 使用 Gmail Trigger 定期偵測新郵件。
- 取得郵件主旨、內文、HTML 或摘要內容。

### 2. AI 郵件內容分析

- 使用大型語言模型分析郵件意圖。
- 將郵件分類至管理部、人事部、資訊部、行銷部、營業部、維修部或客服部。
- 研究 Bi-LSTM、CNN 等自然語言處理模型的分類效果。

### 3. 部門信箱查詢

- 使用 Google Sheets 保存部門名稱與信箱資料。
- 依照 AI 分類結果自動查找對應部門信箱。

### 4. 郵件內容整理與轉寄

- 使用 Python 清理純文字與 HTML 郵件內容。
- 建立標準化郵件主旨及轉寄內容。
- 自動建立草稿或寄送至對應部門。

### 5. LINE 即時通知

- 透過 LINE Messaging API 傳送處理結果。
- 通知內容包含負責部門、郵件主旨與郵件內文。

### 6. 影像理解研究

- 使用 BLIP 進行 Image Captioning、Visual Question Answering 與 Image-Text Matching 實驗。
- 測試 Ollama／本地端模型對影像內容的描述與異常說明能力。

## 系統流程

```text
Gmail 收到新郵件
        ↓
取得郵件主旨與內容
        ↓
AI Agent 判斷負責部門
        ↓
Google Sheets 查詢部門信箱
        ↓
Python 清理並封裝郵件內容
        ↓
建立草稿／自動轉寄郵件
        ↓
LINE 即時通知管理人員
```

## 執行 n8n 工作流

### 前置需求

- 已安裝並可執行 n8n
- Gmail OAuth 2.0 憑證
- Google Sheets OAuth 2.0 憑證
- OpenAI API Key
- LINE Messaging API Channel Access Token

### 匯入方式

1. 啟動 n8n。
2. 進入 Workflows。
3. 選擇 **Import from File**。
4. 匯入：n8n/iMercury最終版.json
5. 重新設定下列節點的個人憑證：
   - Gmail Trigger
   - Gmail
   - OpenAI Chat Model
   - Google Sheets
   - LINE HTTP Request
6. 修改 Google Sheets 文件 ID、工作表名稱與部門信箱資料。
7. 先以測試郵件驗證分類、轉寄及通知結果，再啟用工作流。

> Repository 中不應保存可直接使用的 API Key、OAuth Token、帳號密碼或私人文件 ID。若憑證曾公開，請立即撤銷並重新產生。
